# Codebase Structure

**Analysis Date:** 2026-07-09

## Directory Layout

```
.
├── dot_local/               # User-local executables and data
│   └── bin/                 # Custom scripts (`executable_*`)
├── private_dot_config/      # XDG-compliant app configurations
│   ├── ghostty/             # Terminal emulator config
│   ├── powershell/          # PowerShell profile config
│   ├── private_litellm/     # LiteLLM proxy configuration
│   ├── private_opencode/    # OpenCode and AI agent configuration
│   └── private_tmuxai/      # TmuxAI configuration
├── private_dot_gemini/      # Gemini CLI tool configs
├── private_dot_ssh/         # SSH configurations
└── private_Library/         # macOS specific Library files
    └── LaunchAgents/        # User-level daemons
```

## Directory Purposes

**`private_dot_config/`:**
- Purpose: Application configurations conforming to XDG Base Directory specification.
- Contains: Directories per application (e.g., `ghostty`, `private_opencode`).
- Key files: `private_dot_config/private_opencode/opencode.json.tmpl`

**`dot_local/bin/`:**
- Purpose: Custom user scripts added to `$PATH`.
- Contains: Executable shell scripts.
- Key files: `dot_local/bin/executable_sync-brewfile`, `dot_local/bin/executable_switch-models`

**Root Directory:**
- Purpose: Base dotfiles and chezmoi configurations.
- Contains: `dot_*` files, `run_onchange_*` scripts, manifests (`Brewfile`).
- Key files: `dot_zshrc.tmpl`, `.chezmoi.toml.tmpl`, `run_onchange_install-packages.sh.tmpl`

## Key File Locations

**Entry Points:**
- `.chezmoi.toml.tmpl`: Prompts user for interactive configuration during init.
- `run_onchange_install-packages.sh.tmpl`: Installs packages triggered by changes to `Brewfile`.

**Configuration:**
- `dot_zshrc.tmpl`: Main Zsh shell configuration.
- `dot_tmux.conf.tmpl`: Tmux multiplexer configuration.
- `private_dot_config/private_opencode/opencode.json.tmpl`: Configuration for OpenCode AI tool.

**Core Logic:**
- `dot_zshrc.tmpl`: Defines shell aliases, paths, and environment startup.

**Testing:**
- Not applicable: Standard dotfile repository, no formal test suite present.

## Naming Conventions

**Files:**
- `dot_*`: Translates to `.*` at the root of the home directory (e.g., `dot_zshrc` -> `~/.zshrc`).
- `private_*`: Enforces restricted file permissions (e.g., `0600` or `0700`) during deployment.
- `run_onchange_*`: Scripts executed by chezmoi when their content or a monitored dependency hash changes.
- `executable_*`: Grants executable permissions (`+x`) to the file.
- `*.tmpl`: Handled as Go templates to inject data or apply OS-specific logic.

**Directories:**
- `dot_*`: Translates to `.*` (e.g., `dot_local` -> `~/.local`).
- `private_dot_*`: Translates to `.*` with restricted permissions.

## Where to Add New Code

**New Feature (Shell Aliases / Environment):**
- Primary code: `dot_zshrc.tmpl`

**New Component/Module (Application Config):**
- Implementation: `private_dot_config/<app-name>/`

**Utilities (Custom Scripts):**
- Shared helpers: `dot_local/bin/executable_<script-name>`

**New Packages:**
- macOS: `Brewfile`
- Linux/Arch: `Pacfile`
- Windows: `Winfile`

## Special Directories

**`.planning/`:**
- Purpose: GSD workflow artifacts, codebase mapping, phases, spikes.
- Generated: Partially generated by OpenCode skills.
- Committed: Yes

**`.opencode/`:**
- Purpose: Local installation of OpenCode skills and Node dependencies for AI agents.
- Generated: Yes
- Committed: No (specifically `.opencode/node_modules/` is generated and gitignored)
